# Human vs LLM Risk Attitude Comparison: Data and Code Repository

Welcome to the data and code repository for the Risk Attitude Comparison study. This repository contains all the necessary data, raw LLM outputs, human baseline experiments, analysis scripts, and resulting visualizations used to compare risk-taking behaviors between Human participants and Large Language Models (LLMs).

## Directory Structure Overview

The project is thoughtfully organized into modular directories representing different stages of the data pipeline:

### 1. `HumanExperiments`
Contains everything related to the human baseline data collection and cleaning.
- **`HumanExperimentConfig/`**: Configuration files and interactive assets deployed for the human web experiments.
- **`HumanExperimentRawData&PreResults/`**: Contains the cleaned and processed human response data, most notably `PNAS_Deep_Cleaned.xlsx` which is used by our plotting scripts as the gold standard baseline.
- **`Scripts/`**: Python scripts used exclusively for parsing, cleaning, and preprocessing the raw human experiment JSONs into structured CSV/Excel formats.

### 2. `LLM_RawData`
This folder contains the standardized simulation outputs (100 trials per model) for the tested large language models (e.g., GPT-5.2, Gemini-3.5-Pro, Claude-3.5-Sonnet, DeepSeek-V3.2). This is the finalized dataset used to generate all the comparative reports and figures. 
The data spans the three risk environments tested: DSB (Resource Dynamics), FIP (Financial Investment), and TPB (Triage Scenario).

### 3. `Plotting_Code_And_Data`
Contains the core regression and plotting scripts used to generate the figures for the paper.
- **`KeyinformationExtraction_LLMs.xlsx`**: The consolidated and extracted numerical dataset for the LLMs, built from the raw JSON files.
- **`plot_pnas_figures.py`**: A comprehensive script fitting Ordered Logistic Regression (CLM) models for both humans and LLMs, generating grid S-curves and ABC (Area Between Curves) plots.
- **`plot_human_vs_llm_v3.py`**: Script for exporting ranking and quantitative comparative scatter plots between humans and LLMs.
- **`plot_human_only_baseline.py`**: Generates standalone behavioral S-curves and scatter distributions specifically for the human population.
- **`analyze_results.py`**: Analyzes the raw LLM JSON outputs to calculate the "validity rate" of the models (filtering out timeouts, API errors, and invalid reasoning protocols), generating a bar chart of data quality.

### 4. `Analysis_Results_Human_vs_LLMs`
This folder is the primary output destination for the static `.png` figures generated by the scripts above. It contains:
- `Human_Cleaned_XXX_Scatter_SCurve.png`: Standalone human behavioral models.
- `XXX_ABC_plot.png`: Consolidated S-curves showing the behavioral deviation between specific LLMs and the aggregate baseline.
- `XXX_LLM_Grid_Probabilities.png`: Detailed regressions for individual models.

### 5. `InteractiveVisualization`
Contains dynamic, interactive resources and the final summary markdown report.
- **`v3_human_comparison_report.md`**: The quantitative markdown report listing the final rankings of caution/risk-aversion for Humans and 6 LLMs, calculated using the AUC integral metric.
- **`InteractiveWeb/`**: A suite of self-contained Plotly HTML files allowing you to dynamically explore the S-curves grid and human mappings. You can open `InteractiveWeb/index.html` in any web browser to explore.
- **`Assets/`**: Stores visual assets required for rendering the markdown report.

## How to Review and Run
1. To view the final results, read **`InteractiveVisualization/v3_human_comparison_report.md`** or open **`InteractiveVisualization/InteractiveWeb/index.html`**.
2. To regenerate the static plots, ensure you have Python installed with `pandas`, `seaborn`, `matplotlib`, and `rpy2` (along with an R installation containing the `ordinal` package), then run the scripts within `Plotting_Code_And_Data`.

*Note: All project files have been sanitized and anonymized. Seed traces have been removed to ensure blind review compliance where necessary.*
